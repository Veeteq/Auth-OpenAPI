Authentication

Authentication operations.

User authentication

This operation allows users to authenticate with the system. The user credentials are passed in the request body.

Request Body schema: application/json
required
username
required
string [ 1 .. 255 ] characters ^[a-zA-Z0-9._@-]+$

User's email or username

password
required
string [ 8 .. 128 ] characters

User's password (minimum 8 characters)

Responses

Tenant server

https://{tenant}/api/v1/auth/login

Example server

https://example.com/api/v1/auth/login

User logout

This operation allows users to logout of the system.

Authorizations:
bearerAuth
header Parameters
Set-Cookie
string
Example: refreshToken=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...; Path=/auth/refresh; HttpOnly; Secure; SameSite=Strict; Max-Age=604800

Refresh token cookie (declared for generator/backend). Clients normally send Cookie header; backend expects this header declared.

Refresh user token

Refreshes an expired or expiring authentication token using the refresh token from the Set-Cookie header.

header Parameters
Set-Cookie
string
Example: refreshToken=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...; Path=/auth/refresh; HttpOnly; Secure; SameSite=Strict; Max-Age=604800

Refresh token cookie (declared for generator/backend). Clients normally send Cookie header; backend expects this header declared.

Responses

Tenant server

https://{tenant}/api/v1/auth/refresh

Example server

https://example.com/api/v1/auth/refresh
